# Приложение C. Воспроизводимость и контроль качества

## C1. Принципы воспроизводимости
- Все ключевые численные утверждения в рукописи опираются на пересчитанные сводки из построчных данных.
- Для основной серии использовались согласованные итоговые манифесты стадий.
- Для claim-bearing выводов приоритет отдавался пересчитанным таблицам над legacy-сводками.

## C2. Контроль качества вычислительных артефактов
- Построчное сохранение промежуточных результатов.
- Продолжение прерванных запусков без пересчёта завершённых строк.
- Кэширование результатов автоматического судьи.
- Сопоставление ветвей на совпадающих задачах для исключения артефакта состава выборки.
- Фокусный пересчёт DBR-таблиц из построчных данных.

## C3. Проверка целостности
- По фокусному пересчёту DBR: CRITICAL = 0, HIGH = 0.
- В глобальном аудите фиксировались legacy-несовпадения отдельных старых сводок; они исключены из утверждений, несущих научную нагрузку.

## C4. Ограничения воспроизводимости
- Результаты относятся к режиму inference_recycling_only.
- Основная оценка зависит от автоматического судьи.
- Для ряда сравнений наблюдаются широкие интервалы и чувствительность к seed.
- Ручная проверка 35 пар имеет качественный характер и не является формальной разметкой.
